# HTTP Website Inspector CLI

## Project Goal

This project is a learning-by-practice backend training project.

The goal is to build a command-line tool using Node.js that reads a list of website URLs, sends HTTP requests to them, analyzes their responses, and generates a report.

The project is not only about writing code. It is about understanding backend fundamentals step by step:

* Node.js environment
* npm and package management
* Git and GitHub workflow
* Linux terminal usage
* HTTP requests and responses
* URLs, paths, and query parameters
* JSON files
* Testing with Jest
* Clean project structure

---

# Learning Method

In this project, every stage follows this process:

1. Understand the concept before writing code.
2. Apply the concept in a small practical task.
3. Review the code and the result.
4. Discuss what we learned.
5. Update this README with notes.
6. Commit the change using Git.

The goal is not to copy code.
The goal is to understand why we write every line.

## What is Node.js?

Node.js is a runtime that allows JavaScript to run outside the browser.

Normally, JavaScript runs inside the browser, such as Chrome or Firefox.
But with Node.js, JavaScript can run in the terminal or on a server.

Example:

```js
console.log("Hello from Node.js");
```

Run it with:

```bash
node index.js
```

This means Node.js takes the JavaScript file and executes it.

---

## What is npm?

npm is the package manager for Node.js.

We use npm to:

* Initialize a project
* Install packages
* Manage dependencies
* Run project scripts

Example:

```bash
npm init -y
```

This creates a `package.json` file.

Example:

```bash
npm install jest --save-dev
```

This installs Jest as a development dependency.

---

## What is nvm?

nvm means Node Version Manager.

It allows us to install and switch between different Node.js versions.

Useful commands:

```bash
nvm current
```

Shows the current Node version.

```bash
nvm use
```

Uses the Node version written in `.nvmrc`.

```bash
nvm install --lts
```

Installs the latest LTS version of Node.js.

---

## What is package.json?

`package.json` is the identity file of a Node.js project.

It stores information such as:

* Project name
* Project version
* Scripts
* Dependencies
* Development dependencies

Example:

```json
{
  "name": "http-website-inspector-cli",
  "version": "1.0.0",
  "type": "module",
  "scripts": {
    "start": "node src/index.js",
    "test": "jest"
  }
}
```

---

## What is package-lock.json?

`package-lock.json` stores the exact versions of installed packages.

We do not edit this file manually.

It is useful because it helps everyone install the same dependency versions.

This file should be committed to Git.

---

## What is node_modules?

`node_modules` is the folder that contains installed packages.

We do not upload `node_modules` to GitHub because:

* It is usually very large.
* It can be recreated using `npm install`.
* It is not source code.

That is why we add it to `.gitignore`.

---

## What is .gitignore?

`.gitignore` tells Git which files or folders to ignore.

For this project, we use:

```txt
node_modules/
.env
report.json
```

Meaning:

* `node_modules/` should not be tracked.
* `.env` should not be tracked because it may contain secrets.
* `report.json` should not be tracked because it is generated output.

---

## What is .nvmrc?

`.nvmrc` stores the Node.js version used in this project.

Example:

```txt
v24.16.0
```

When entering the project later, we can run:

```bash
nvm use
```

This makes sure we are using the correct Node.js version.

## Git Workflow Used

We use Git to track project history.

Basic flow:

```bash
git status
git add .
git commit -m "Message"
git push
```

Meaning:

1. `git status` checks the current state.
2. `git add` stages files.
3. `git commit` saves a snapshot locally.
4. `git push` uploads commits to GitHub.

---

## Important Lesson from Stage 0

Before writing code, a backend developer should make sure the environment is ready.

The basic questions are:

* Am I inside the correct folder?
* Is Node.js installed?
* Is npm working?
* Is the correct Node version active?
* Is Git tracking the project?
* Is `.gitignore` protecting unnecessary files?
* Is the project connected to GitHub?
